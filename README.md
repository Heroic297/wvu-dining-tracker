# WVU Dining Tracker

A full-stack nutrition tracking web application for West Virginia University students. Log dining hall meals, track custom foods with USDA + Groq AI nutrition estimation, manage diet goals, connect Garmin/Fitbit for real activity data, and run a powerlifting meet water-cut plan.

**Live stack:** React + Express + PostgreSQL (Supabase) + Drizzle ORM + Tailwind CSS + shadcn/ui

---

## Features

- **Authentication** — Email/password with JWT tokens; password-reset via Supabase
- **WVU Dining Hall Menus** — Auto-scrapes Café Evansdale, Hatfields, Summit Café daily at 6 AM EST via DineOnCampus API
- **Meal Logging** — Select dining hall + meal, add items, adjust servings, track macros
- **Custom Foods** — Search USDA FoodData Central; Groq Qwen 3 32B AI fallback for unknown foods
- **TDEE / BMR Calculations** — Mifflin-St Jeor equation with activity level multipliers
- **Diet Planning** — Deficit/surplus targets, training vs. rest day adjustments
- **Powerlifting Mode** — Meet date tracking, 7-day water cut protocol
- **Wearable Integration** — Garmin & Fitbit OAuth2 for real daily burn data
- **Weight Tracking** — Log daily weight, view trend charts

## Local Development

### Prerequisites
- Node.js 20+
- PostgreSQL database (Supabase free tier recommended)
- Groq API key (free at [console.groq.com](https://console.groq.com))

### 1. Clone and install

```bash
git clone https://github.com/YOUR_USERNAME/wvu-dining-tracker.git
cd wvu-dining-tracker
npm install
```

### 2. Configure environment

Copy `.env.example` to `.env` and fill in values:

```bash
cp .env.example .env
```

### 3. Run database migrations

```bash
npm run db:push
```

### 4. Start dev server

```bash
npm run dev
```

Open http://localhost:5000

## Environment Variables

Create a `.env` file in the project root:

```env
# ─── Database ───────────────────────────────────────────────────────────────
# PostgreSQL connection string (Supabase or any Postgres)
DATABASE_URL=postgresql://postgres:[password]@db.[project].supabase.co:5432/postgres

# ─── Authentication ──────────────────────────────────────────────────────────
# Secret for signing JWT tokens — generate with: openssl rand -hex 32
JWT_SECRET=your-very-long-random-secret-here

# Optional: Supabase JWT secret (for validating Supabase-issued JWTs)
SUPABASE_JWT_SECRET=your-supabase-jwt-secret

# Session secret
SESSION_SECRET=another-random-secret

# ─── Groq AI ─────────────────────────────────────────────────────────────────
# Get from: https://console.groq.com
GROQ_API_KEY=gsk_...

# ─── USDA FoodData Central ───────────────────────────────────────────────────
# Free API key at: https://fdc.nal.usda.gov/api-guide.html
# Omit to use DEMO_KEY (30 req/hr limit)
USDA_API_KEY=DEMO_KEY

# ─── Fitbit OAuth2 ───────────────────────────────────────────────────────────
# Register at: https://dev.fitbit.com/apps/new
# Callback URL: https://your-app.onrender.com/api/wearables/fitbit/callback
FITBIT_CLIENT_ID=
FITBIT_CLIENT_SECRET=
FITBIT_REDIRECT_URI=http://localhost:5000/api/wearables/fitbit/callback

# ─── Garmin OAuth2 ───────────────────────────────────────────────────────────
# Register at: https://developer.garmin.com/gc-developer-program/overview/
# Callback URL: https://your-app.onrender.com/api/wearables/garmin/callback
GARMIN_CLIENT_ID=
GARMIN_CLIENT_SECRET=
GARMIN_REDIRECT_URI=http://localhost:5000/api/wearables/garmin/callback

# ─── Background Job Security ─────────────────────────────────────────────────
# Random secret for /api/jobs/* endpoints (optional external cron trigger)
CRON_SECRET=your-cron-secret

# ─── Server ──────────────────────────────────────────────────────────────────
PORT=5000
NODE_ENV=development
```

## Supabase Setup

1. Create a free project at [supabase.com](https://supabase.com)
2. Go to **Settings → Database** and copy the connection string (URI mode)
3. Use that as `DATABASE_URL`
4. For email auth: **Authentication → Providers → Email** — enable it
5. For Google auth: **Authentication → Providers → Google** — configure OAuth app
6. Copy **Project Settings → API → JWT Secret** to `SUPABASE_JWT_SECRET`

## Render Deployment

### 1. Create a Render Web Service

Go to [dashboard.render.com](https://dashboard.render.com) → **New** → **Web Service** → connect your GitHub repo.

### 2. Configure the service

| Setting | Value |
|---------|-------|
| **Environment** | Node |
| **Build Command** | `npm install && npm run build` |
| **Start Command** | `npm run start` |
| **Branch** | `main` |

### 3. Add environment variables

In your Render service → **Environment**, add all variables from `.env.example` with production values:
- `NODE_ENV=production`
- `JWT_SECRET` — a long random string
- `GROQ_API_KEY` — your Groq key
- `DATABASE_URL` — your Supabase connection string
- Fitbit/Garmin callback URLs pointing to your Render domain

### 4. Deploy

Render auto-deploys on every push to `main`. You can also trigger a manual deploy from the dashboard.

### 5. Rollback

In the Render dashboard → your service → **Events** tab, click any previous successful deploy and hit **Redeploy** to instantly roll back.

### 6. Scheduled scraping job (optional external trigger)

Use Render's **Cron Jobs** (under your project) or any external scheduler to hit the scrape endpoint daily:
- URL: `POST https://your-app.onrender.com/api/jobs/scrape`
- Header: `x-cron-secret: YOUR_CRON_SECRET`
- Schedule: `0 11 * * *` (6 AM EST = 11 AM UTC)

> **Note:** The app also runs an internal node-cron scheduler automatically, so the external cron is optional redundancy.

### 7. Update OAuth callback URLs

Once deployed, update Fitbit and Garmin app settings to point callbacks to:
- `https://your-app.onrender.com/api/wearables/fitbit/callback`
- `https://your-app.onrender.com/api/wearables/garmin/callback`

And update the env vars accordingly.

1. Apply at [developer.garmin.com](https://developer.garmin.com/gc-developer-program/overview/)

3. Note: Garmin's approval process can take a few days

## Fitbit Developer Setup

1. Go to [dev.fitbit.com](https://dev.fitbit.com/apps/new)
2. Create a **Personal** application
3. Set OAuth 2.0 Application Type: **Personal**
4. Callback URL: your Render URL `/api/wearables/fitbit/callback`

## Database Schema

Key tables:
- `users` — Auth + profile + TDEE settings + powerlifting config
- `wearable_tokens` — OAuth tokens for Garmin/Fitbit
- `daily_activity` — Synced daily burn data from wearables
- `dining_locations` — Seeded with WVU dining halls
- `dining_menus` — Scraped menu instances (location × date × meal_type)
- `dining_items` — Individual food items per menu
- `nutrition_cache` — USDA + AI nutrition lookup cache
- `user_meals` — User meal sessions (date × meal_type)
- `user_meal_items` — Individual logged items per meal
- `weight_log` — Daily weight entries

## Updating the Scraper

The WVU dining data comes from DineOnCampus's API (`api.dineoncampus.com`). If it changes:

1. Open https://dineoncampus.com/wvu in Chrome
2. Open DevTools → Network tab → filter for `api.dineoncampus.com`
3. Look for `location/*/periods` requests
4. Update `WVU_LOCATION_IDS` in `server/scraper.ts` with new location IDs
5. Update the parsing logic if the response format changed

## Key API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/auth/register` | Register new user |
| POST | `/api/auth/login` | Login |
| GET | `/api/auth/me` | Current user |
| PATCH | `/api/user/profile` | Update profile/settings |
| GET | `/api/dashboard` | Today's summary |
| GET | `/api/targets?date=` | Daily calorie + macro targets |
| GET | `/api/dining/locations` | List dining halls |
| GET | `/api/dining/menu?locationSlug=&date=&mealType=` | Menu items |
| GET | `/api/nutrition/lookup?q=` | USDA + AI nutrition lookup |
| GET | `/api/meals?date=` | Get meals for a date |
| POST | `/api/meals` | Create a meal log entry |
| POST | `/api/meals/:id/items` | Add food item to meal |
| DELETE | `/api/meal-items/:id` | Remove item |
| POST | `/api/weight` | Log weight |
| GET | `/api/weight` | Weight history |
| GET | `/api/wearables/fitbit/connect` | Get Fitbit OAuth URL |
| GET | `/api/wearables/garmin/connect` | Get Garmin OAuth URL |
| DELETE | `/api/wearables/:source` | Disconnect wearable |
| POST | `/api/jobs/scrape` | Trigger scrape (cron) |
